\chapter{Introduction}

\section{Problem Statement}
Optical Character Recognition (OCR) is a well-established field that has achieved high accuracy in processing standard, high-resolution documents. However, applying these techniques to severely constrained spatial grids—specifically 16x16 binary pixel matrices—presents notable practical challenges. At this extreme low resolution, the morphological properties of text are highly degraded. Continuous curves and diagonal strokes suffer from severe aliasing, leading to a massive loss of geometric information.

Beyond the physical limitations of the resolution, there is a critical domain gap between synthetic training data and real-world human input. Standard datasets (such as the Unifont dataset) provide characters with perfect, 1-pixel-thick static lines. Conversely, when users interact with a graphical interface—drawing characters freehand using a computer mouse or trackpad—the inputs are highly erratic. Human-generated strokes exhibit unpredictable variations in thickness, localized elastic distortions, angular skewing (shear), and arbitrary spatial offsets. A Convolutional Neural Network (CNN) trained exclusively on clean, synthetic fonts often experiences a drastic drop in accuracy when exposed to these noisy, real-world trajectories because it has not learned to generalize against such human factors.

Additionally, conventionally deployed machine learning models are static, meaning they do not possess an intrinsic mechanism to correct their own misclassifications after deployment. If a model consistently misinterprets a specific idiosyncratic handwriting style, it will continue to fail without further updates. Exploring a dynamic approach, where user corrections are systematically utilized as a feedback mechanism, provides a practical method to incrementally improve model robustness in real-world scenarios.

\section{Objectives}
This project focuses on implementing and evaluating an adaptive 16x16 OCR system by integrating a Convolutional Neural Network with an Active Learning feedback loop. As a practical application of computer vision and machine learning concepts, the primary objectives are defined as follows:
\begin{itemize}
    \item \textbf{Implement a Baseline CNN Architecture:} To construct and train a lightweight Convolutional Neural Network tailored for extracting morphological features (such as loops and stroke intersections) from low-resolution 16x16 binary matrices.
    \item \textbf{Apply Data Augmentation and Preprocessing:} To mitigate the domain gap between synthetic fonts and human handwriting by utilizing established image processing techniques. This includes applying mathematical transformations such as directional stroke thickening, edge jitter, affine transformations, and elastic deformations. Furthermore, an auto-centering preprocessing algorithm is implemented to normalize spatial offsets in raw user inputs.
    \item \textbf{Develop an Interactive GUI with a Feedback Loop:} To build a graphical user interface that facilitates real-time character drawing and prediction. A core component of this system is the implementation of a data flywheel mechanism designed to collect user-corrected misclassifications. This collected data is then used to periodically fine-tune the model, allowing the system to gradually adapt to diverse, real-world writing patterns.
\end{itemize}

\chapter{Theoretical Background}
\label{sec:theoretical-background}

\section{Problem Formulation}
The character recognition task on low-resolution grids, augmented by an active learning feedback loop, can be modeled mathematically in two distinct phases: static optimization and dynamic adaptation.

\subsection*{Data Space and Static Optimization}

Let $\mathcal{X}$ denote the input space of $16 \times 16$ binary image matrices, where an individual input is defined as $x \in \{0, 255\}^{16 \times 16}$. Let $\mathcal{Y} = \{1, 2, \dots, K\}$ represent the label space, with $K$ being the total number of character classes. The initial synthetic training dataset is defined as $D_{train} = \{(x_1, y_1), (x_2, y_2), \dots, (x_N, y_N)\}$, where $N$ is the total number of generated samples.

The objective of the Convolutional Neural Network (CNN) is to learn a mapping function $f_\theta: \mathcal{X} \rightarrow \mathbb{P}(\mathcal{Y})$, parameterized by a set of weights $\theta$, where $\mathbb{P}(\mathcal{Y})$ is the predicted probability distribution over the $K$ classes. The optimal parameters $\theta^*$ are obtained by minimizing the objective function over $D_{train}$ using the Categorical Cross-Entropy loss:

\begin{equation}
\theta^* = \arg\min_{\theta} \left[ \frac{1}{N} \sum_{i=1}^{N} \mathcal{L}(f_\theta(x_i), y_i) \right]
\end{equation}

For a single sample $(x_i, y_i)$, the loss function $\mathcal{L}$ is defined as:

\begin{equation}
\mathcal{L}(\hat{y}_i, y_i) = - \sum_{j=1}^{K} \mathbb{1}(y_i = j) \log(\hat{y}_{i,j})
\end{equation}

where $\hat{y}_i = f_\theta(x_i)$ is the output vector from the Softmax activation layer, and $\mathbb{1}(\cdot)$ is the indicator function.

\subsection*{2. Dynamic Adaptation via Active Learning}

To address the domain shift between synthetic data $\mathcal{P}_{aug}(x, y)$ and real-world human inputs $\mathcal{P}_{real}(x, y)$, the system incorporates a temporal feedback loop. At a given time step $t$, the system collects a set of user-corrected misclassifications, denoted as $D_{feedback}^{(t)} \sim \mathcal{P}_{real}(x, y)$.

The training dataset is incrementally updated as $D^{(t+1)} = D^{(t)} \cup D_{feedback}^{(t)}$. The model undergoes a fine-tuning process to update its weights to $\theta_{t+1}^*$, utilizing $D_{feedback}$ as a regularizing force to guide the model toward the true real-world distribution without experiencing catastrophic forgetting.



\section{Data Augmentation for Low-Resolution Binary Images}
Data augmentation is a regularization technique used to artificially expand the size and diversity of the training dataset. In standard computer vision tasks, techniques such as Gaussian blur, color jittering, or global salt-and-pepper noise are widely utilized. However, applying these conventional methods to $16 \times 16$ binary grids is detrimental, as continuous noise easily destroys the discrete topological structure of small characters. 

To effectively simulate human handwriting variations while preserving the structural integrity of the binary characters, a specialized augmentation pipeline is required. This pipeline relies on two fundamental categories of transformations: Geometric (Affine) Transformations and Morphological Operations.

\subsection*{Geometric (Affine) Transformations}
Affine transformations modify the geometric structure of the image by mapping pixel coordinates to new positions while preserving collinearity (straight lines remain straight) and ratios of distances. 

\begin{itemize}
    \item \textbf{Translation (Shift):} Freehand drawing often results in characters that are not perfectly centered. Translation mathematically shifts all pixel coordinates by a vector $(d_x, d_y)$. This forces the Convolutional Neural Network to learn spatial invariance, ensuring that the model recognizes a character based on its shape rather than its absolute position on the grid.
    \item \textbf{Rotation:} To simulate the varying angles and orientations of human handwriting, a 2D rotation matrix is applied to the image coordinates. For low-resolution grids, rotation angles must be tightly constrained; otherwise, the interpolation process during coordinate mapping can cause severe aliasing and stroke fragmentation.
    \item \textbf{Shear Transformation:} Shear transforms rectangles into parallelograms by displacing each pixel proportionally to its distance from an axis. Mathematically, a horizontal shear displaces the $x$-coordinate based on the $y$-coordinate. This perfectly simulates slanted or italicized handwriting styles without altering the fundamental height of the characters.
\end{itemize}

\subsection*{Morphological Operations}
Unlike affine transformations that move the entire object in space, morphological operations alter the intrinsic structure and thickness of the strokes based on set theory and localized pixel neighborhoods.

\begin{itemize}
    \item \textbf{Directional Thickening:} Standard morphological dilation expands the boundaries of foreground pixels uniformly in all directions. On a severely constrained $16 \times 16$ grid, uniform dilation inevitably causes closely spaced strokes (such as the inner loops of 'e', 'a', or '6') to merge into unrecognizable solid blocks. To simulate the thicker strokes of a computer mouse or a bold stylus, \textit{directional thickening} is employed. This operation calculates the maximum value between the original image and a translated copy of itself in a single Cartesian direction (e.g., purely horizontal or purely vertical). This method safely increases stroke weight by one pixel while strictly preserving the internal topology.
    \item \textbf{Edge Jitter (Boundary Noise):} Global random noise is an unrealistic representation of digital drawing on a clean canvas. Human drawing inaccuracies typically manifest as micro-tremors, resulting in jagged or uneven stroke boundaries. To mathematically simulate this, morphological dilation and erosion are first utilized to isolate the exact outer and inner contours of the character's strokes. Stochastic binary noise is then applied exclusively to these extracted boundaries. This technique accurately models realistic hand tremors (adding stray pixels to the outer edge or removing pixels from the inner edge) without corrupting the core structure of the character.
\end{itemize}

\chapter{Methodology and System Implementation}
\label{sec:methodology}

This chapter details the practical implementation of the theoretical concepts discussed in Chapter 2. It outlines the complete pipeline of the proposed OCR system, from the initial parsing and augmentation of the synthetic dataset to the architectural design of the CNN, and finally, the deployment of the interactive Graphical User Interface (GUI) with real-time post-processing.



\section{Data Preprocessing and Augmentation Pipeline}
The foundation of the model's robustness relies on a carefully engineered data generation pipeline. The initial dataset is provided in a hexadecimal text format, where each line corresponds to a character label and a 256-bit hexadecimal string. The preprocessing module sequentially reads the \texttt{.hex} file, decodes the 256-bit string into a binary array, and reshapes it into a $16 \times 16$ NumPy matrix. Pixels are mapped to discrete binary values, utilizing 0 for the background and 255 for the foreground character stroke.

To bridge the domain gap between these static, 1-pixel-thick synthetic fonts and erratic human handwriting, a randomized augmentation pipeline is applied. For every base character, 99 additional variant samples are generated (totaling 100 samples per class) using a combination of stochastic transformations. The parameters and application probabilities for each transformation were strictly calibrated based on the spatial constraints of the $16 \times 16$ grid.

\textbf{Directional Thickening:} Applied with a probability of 0.6, this transformation shifts the character exactly one pixel in a randomly selected direction (up, down, left, or right) and merges it with the original image using a maximum logical operator. This method simulates the thicker strokes naturally produced by a computer mouse or a bold stylus. A uniform morphological dilation was deliberately avoided because expanding strokes in all directions on a tight $16 \times 16$ grid would cause the internal spaces of dense characters (such as 'e', 'a', or 'B') to merge into solid, unrecognizable blocks. The 60\% application rate ensures the dataset contains a balanced mix of thin and thick variants.

\textbf{Rotation:} Rotation is applied with a 0.6 probability, where the angle is uniformly sampled from $-12^\circ$ to $12^\circ$. While human handwriting involves various tilt angles, the rotation parameter is tightly constrained to a maximum of $\pm 12^\circ$. Empirical observation demonstrates that rotating discrete pixels beyond this threshold on low-resolution grids causes severe aliasing and stroke fragmentation during the coordinate mapping process. This strict limit preserves stroke continuity while still teaching the network rotational invariance.

\textbf{Shear Transformation:} To model italicized or slanted writing styles, a shear transformation is applied with a 0.5 probability using a shear factor uniformly distributed between $-0.25$ and $0.25$. This maximum shear factor was specifically chosen to provide a noticeable slant without displacing the top and bottom edges of the characters outside the strict $16 \times 16$ bounding box.

\textbf{Translation (Shift):} Because users rarely draw characters perfectly centered on the canvas, translation is given the highest application probability of 0.7 to force the CNN to learn spatial invariance. The spatial shifts are randomly selected from $-2$ to $2$ pixels along both the x and y axes. The maximum shift is intentionally capped at $\pm 2$ pixels; shifting further would cause wider characters to be cropped at the canvas boundaries, leading to detrimental information loss.

\textbf{Edge Jitter Noise:} Global salt-and-pepper noise is unrealistic for digital canvas drawing. Instead, a custom edge jitter noise is applied with a 0.5 probability, strictly targeting the character's boundaries. This involves a 25\% probability of adding noise pixels to the outer boundary (simulating protruding artifacts) and a 15\% probability of removing pixels from the inner boundary (simulating chipped strokes). These application rates are kept intentionally low to simulate the micro-tremors of a human hand operating a mouse, ensuring the noise roughens the edges without completely breaking the fundamental topology of the character.

\section{Custom CNN Architecture for 16x16 Inputs}
Given the severely constrained spatial dimensions of the input data ($16 \times 16 \times 1$), deploying a deep, standard CNN architecture (such as ResNet or VGG) is impractical. Such deep networks would rapidly reduce the spatial dimensions to zero through successive pooling layers and would heavily overfit the small-scale features of binary characters. Therefore, a custom, lightweight CNN architecture, encapsulated in the \texttt{StandardCNN} class, was designed from scratch to optimize feature extraction while maintaining computational efficiency.

\subsection*{Input Normalization}
Before propagating through the network, the raw $16 \times 16$ input matrix undergoes a mandatory normalization step. The pixel intensities are linearly scaled from their original grayscale range of $[0, 255]$ to a zero-centered range of $[-0.5, 0.5]$ using the linear transformation \texttt{(image / 255.0) - 0.5}. 

This zero-centering process is mathematically critical for the stability and efficiency of the gradient descent algorithm. If the input values were strictly positive (e.g., scaled only to $[0, 1]$ or left as $[0, 255]$), the local gradients of the weights during backpropagation, calculated as $\frac{\partial L}{\partial W_i} = \frac{\partial L}{\partial Z} X_i$, would all share the exact same sign as the upstream gradient $\frac{\partial L}{\partial Z}$. Consequently, all weights connected to a specific neuron would be forced to either increase or decrease simultaneously during a single update step. This restriction severely limits the optimizer's freedom of movement, leading to highly inefficient, zigzagging optimization trajectories across the loss landscape.

By shifting the input domain to $[-0.5, 0.5]$, the input vector contains both positive (foreground strokes) and negative (background) values. This completely decouples the weight updates, allowing the optimizer to adjust individual weights in opposite directions simultaneously, thereby drastically accelerating the convergence rate. Furthermore, bounding the input variance near zero prevents the activation values from exploding during successive matrix multiplications, effectively mitigating the risk of numerical overflow (e.g., \texttt{NaN} outputs) during the Softmax exponentiation in the final layer.

\subsection*{First Convolutional Block}
The first block focuses on extracting low-level geometric primitives, such as horizontal, vertical, and diagonal strokes.
\begin{itemize}
    \item \textbf{Conv2D Layer:} The network applies 16 filters of size $3 \times 3$ over the single-channel input. A $3 \times 3$ kernel serves as the smallest possible window to capture structural relationships like corners or intersections, while 16 filters provide adequate capacity to learn primary edge variations without overwhelming the model with redundant parameters. This convolution operates without zero-padding (valid padding) to avoid introducing artificial background zeros at the image borders, subsequently reducing the spatial dimensions from $16 \times 16$ to $14 \times 14$.
    \item \textbf{ReLU Activation:} A Rectified Linear Unit (ReLU) is applied to introduce non-linearity by capping negative values to zero.
    \item \textbf{MaxPool2D Layer:} A non-overlapping $2 \times 2$ Max Pooling operation is applied to downsample the feature maps from $14 \times 14$ to $7 \times 7$. This operation isolates the most prominent structural features, provides local translation invariance, and drastically reduces the computational load for the subsequent layers.
\end{itemize}

\subsection*{Second Convolutional Block}
The second block is designed to combine the low-level strokes extracted by the first block into more complex, character-defining structures, such as closed loops or crossing intersections.
\begin{itemize}
    \item \textbf{Conv2D Layer:} This layer takes the 16 channels from the previous block and applies 32 filters of size $3 \times 3$. Increasing the depth to 32 filters allows the network to represent higher-level abstractions within the much smaller spatial dimension. Using valid convolution, the spatial dimensions are further reduced from $7 \times 7$ to $5 \times 5$.
    \item \textbf{ReLU Activation:} A second ReLU activation is applied to maintain non-linearity.
    \item \textbf{MaxPool2D Layer:} Another $2 \times 2$ pooling layer is utilized to condense the feature maps from $5 \times 5$ down to a highly compressed $2 \times 2$ spatial grid.
\end{itemize}

\subsection*{Fully Connected (Dense) Classifier}
The final phase of the architecture translates the 3D feature maps into a 1D vector to perform global reasoning and classification.
\begin{itemize}
    \item \textbf{Flatten Layer:} The $2 \times 2 \times 32$ feature maps are flattened into a single 1D vector consisting of exactly 128 elements.
    \item \textbf{First Dense Layer:} A fully connected layer maps the 128 input nodes to 128 hidden nodes, followed by a ReLU activation. This layer acts as a non-linear combination unit, aggregating the spatial features across the entire $2 \times 2$ grid to understand the global topology of the character.
    \item \textbf{Output Dense Layer:} The final fully connected layer maps the 128 hidden nodes down to the exact number of character classes defined in the dataset.
    \item \textbf{Softmax Activation:} The output logits are passed through a Softmax function, converting them into a normalized probability distribution where the sum of all class probabilities equals 1. To ensure numerical stability during the backward pass and prevent overflow, the custom Softmax implementation safely subtracts the maximum logit before exponentiation.
\end{itemize}

\subsection*{Weight Initialization and Loss Calculation}
\begin{itemize}
    \item \textbf{He Initialization:} To mitigate the vanishing gradient problem, the weights in both Conv2D and Dense layers are explicitly initialized using a scaled Gaussian distribution multiplied by $\sqrt{2.0 / \text{input\_size}}$. This conforms to the standard He initialization scheme, which is mathematically optimized for networks utilizing ReLU activations.
    \item \textbf{Categorical Cross-Entropy Loss:} During the training step, the model evaluates its predictions using the cross-entropy loss function. A small epsilon value of $1 \times 10^{-9}$ is systematically added to the predicted probability before taking the logarithm. This addition prevents critical mathematical errors, such as calculating the log of zero, particularly when the network is highly unconfident in its early epochs.
\end{itemize}

AUC of 0.820, while the treatment model reached an AUC of 0.834, indicating strong
discriminative ability within each subgroup and reliable prediction of churn likelihood.

Regarding uplift-specific metrics, the model obtained a Qini coefficient of 4145.49,
which confirms that it successfully differentiates between customers who are more
likely to respond positively to retention offers versus those who are not. The Area
Under the Uplift Curve (AUUC) was slightly negative (–5841.25), suggesting minor
inconsistencies in uplift ranking or calibration. This limitation is likely caused by the
randomly generated treatment labels in the dataset rather than true causal responses.

Overall, despite the small uplift magnitude, the model is able to identify customer
subgroups with differing incremental responses to promotional offers. Additional
improvements are expected if real treatment labels become available or if causal
models with better uplift calibration are introduced.

\chapter{Conclusions}
\label{sec:conclusion}

This report set out to address a central question for subscription-based telecommunications businesses: how to predict which customers are likely to churn, and how to design cost–effective retention strategies that focus resources on the right individuals. In Chapter~\ref{sec:problem-formulation}, we formulated churn management as a structured decision problem involving three layers: estimating individual churn risk, estimating the impact of a retention offer (uplift), and translating these estimates into financial outcomes. Chapter~\ref{sec:methodology} then presented the modeling framework for churn prediction, uplift estimation using a two-model approach, and a simple financial simulation linking uplift to net savings under explicit assumptions about customer lifetime value and incentive costs.

\subsection*{Key Findings}

A first key conclusion is that \textbf{churn is predictable rather than random}. The customer-level data show systematic relationships between churn behavior and a small set of intuitive drivers. In particular, contract type, tenure, and payment method play a central role:
\begin{itemize}
    \item customers on \emph{month-to-month} contracts exhibit substantially higher churn risk than those on one–year or two–year contracts;
    \item customers with \emph{short tenure} (newer customers) are more likely to leave than long–standing subscribers;
    \item customers paying by \emph{electronic check} are more prone to churn than those using more stable payment methods such as credit card or bank transfer.
\end{itemize}
These patterns confirm that churn is driven by identifiable contractual and behavioral factors, and that a reasonably simple supervised model can capture a meaningful share of the variation in churn risk at the individual level.

At the same time, the analysis highlights that \textbf{prediction alone is not sufficient} for sound business decisions. A high churn probability tells us who is likely to leave under current conditions, but it does not answer the more operational question: \emph{what should we do about it?} If retention offers are directed mechanically at all ``high–risk'' customers, the company may end up:
\begin{itemize}
    \item subsidizing \emph{loyals} who would have stayed anyway even without an offer; and
    \item wasting incentives on \emph{lost causes} who will churn regardless of any discount or promotion.
\end{itemize}
Both situations erode profitability, even if the churn model is statistically accurate.

This motivates the third key conclusion: \textbf{uplift is the right lens for retention strategy}. By explicitly modeling the difference between churn probabilities with and without an offer, the uplift framework focuses attention on the \emph{persuadables}—customers whose decision is actually changed by the incentive. Only in this segment does a retention offer create genuine incremental value. When uplift scores are combined with simple financial assumptions (customer lifetime value and offer cost), it becomes possible to distinguish between:
\begin{itemize}
    \item customers for whom the expected retained CLV exceeds the cost of the offer (positive expected net savings); and
    \item customers for whom the offer is not economically justified.
\end{itemize}
Conceptually, this closes the loop from prediction to decision: not only identifying who is at risk, but also who should be targeted to achieve positive return on investment.

\subsection*{Managerial Implications}

From a managerial perspective, the main implication is that churn and retention should be treated as an integrated, data–driven process rather than as separate reporting and campaign activities. A practical implementation of the proposed framework would provide front–line teams with:
\begin{itemize}
    \item a \emph{churn scorecard} that ranks customers by risk;
    \item an \emph{uplift score} or ``next best action'' signal that indicates where offers are most likely to change behavior; and
    \item a simple view of expected financial impact (retained CLV, incentive cost, and net savings) for different targeting thresholds.
\end{itemize}
Such tools can support more disciplined decisions about whom to contact, what kind of offer to extend, and how to balance campaign volume against budget constraints.

\subsection*{Limitations and Future Work}

The current report uses a simulated treatment assignment and a stylized financial model. As a result, the uplift estimates and net savings are \emph{scenario–based} rather than measured causal effects. This leads directly to the most important next steps:
\begin{itemize}
    \item \textbf{Run an A/B test pilot.} A controlled experiment on a small, representative customer cohort is essential to validate the uplift model in practice. By randomly assigning real offers to a treatment group and comparing outcomes to a control group, the telecom operator can measure true incremental retention and refine the model accordingly.
    \item \textbf{Integrate and automate in operational systems.} To create ongoing value, the churn scorecard and uplift–based ``next best action'' should be embedded into the call center’s CRM or customer management platform. This would allow agents to see risk and recommended actions in real time and to record actual responses, feeding back new data to continuously improve the models.
\end{itemize}

Beyond these immediate next steps, future work could explore richer offer strategies (e.g., multiple offer types with different costs and effects), more granular CLV estimation, and alternative uplift modeling techniques. However, even in its current form, the framework presented in this report provides a coherent path from raw customer data to actionable, financially grounded retention decisions. It demonstrates that, when combined with uplift modeling and basic economics, churn prediction can move from descriptive reporting to a practical tool for managing subscription relationships more profitably.
